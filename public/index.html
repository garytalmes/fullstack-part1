Fullstack Explorer Part I

With your teammates, take a look at the source code running this site and see if you can answer the questions below. Some of these questions may require some Googling!

  • How is the form on the contact page sending data to the server?
  • Where in the server.js file is that form data being received?
  • When the form data is received, what is the first thing the code checks for? And what does it do if that check fails?
  • In some of the response code you will see res.status. What does that do?
  • If the user submits valid data, what are we doing with that data?
  • Notice that the readFile and writeFile methods are being called differently than you may have seen before? What's up with that?
  • What logic am I using to generate the name of the file which is written by the server?
  • How are we handling the result that the server sends back to the contact form after submission?

If you finish early, try any of the following:

  • Add another field to the contact form and make sure that data is also written to the file on the server.
  • Write a catch-all route in case someone tries to go to a page that doesn't exist.
  • Create another html page and a route which will serve it. When the page hits the browser, it should also load a Javascript file. That Javascript file should make a GET request back to the server for some data. Check the data folder in the server code. Can you grab that data, send it to the browser, and have the browser display it?
